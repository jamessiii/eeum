# 이음 설정 반자동화 구상

## 배경

- 설정 화면은 사용자가 처음 진입했을 때 빠르게 기본 구조를 갖추도록 도와야 한다.
- 모든 것을 직접 결정하게 하기보다, 확실한 기본 연결은 자동으로 제안하거나 적용하는 쪽이 맞다.
- 다만 자동화는 사용자를 막는 확인 팝업이 아니라, 결과를 보여주고 필요하면 되돌릴 수 있는 방향이어야 한다.

## 원칙

1. 확실한 것은 자동 적용한다.
2. 애매한 것만 짧게 확인받는다.
3. 막는 팝업보다 `자동 적용 + 되돌리기`를 우선한다.
4. 같은 타입의 두 번째 계좌부터는 조용히 자동 연결하지 않는다.

## 현재 구현 기준

### 1. `daily` 계좌 자동 연결

- 첫 `daily` 계좌가 생성되거나 기존 계좌가 처음 `daily` 로 바뀌면 생활비 계열 카테고리와 자동 연결한다.
- 이미 같은 타입의 계좌가 있으면 추가 자동 연결은 하지 않는다.

### 2. `loan` 계좌 자동 연결

- 첫 `loan` 계좌가 생성되거나 기존 계좌가 처음 `loan` 으로 바뀌면 대출/부채 계열 카테고리와 자동 연결한다.
- 이미 같은 타입의 계좌가 있으면 추가 자동 연결은 하지 않는다.

## 다음 자동화 범위

### 1. usageType 기반 자동 세팅

- 첫 `daily` 계좌
  - 생활 계열 카테고리 자동 연결
- 첫 `loan` 계좌
  - 대출/부채 계열 카테고리 자동 연결
- 첫 `card_payment` 계좌
  - 카드 납부 계좌 기본 후보로 제안 또는 연결
- 첫 `salary` 계좌
  - 수입 기준 계좌 기본 후보로 제안 또는 연결

## 제외 규칙

- `shared` 계좌 자동화는 더 이상 진행하지 않는다.
- 이 앱에는 `공동지출`, `정산`, `N빵` 개념이 없다.
- 사람 간 분배/상환 흐름을 위한 기본 계좌 후보를 만드는 방향은 현재 제품 규칙과 맞지 않는다.

## 제안 UX

1. 사용자가 계좌/카드를 만든다.
2. 확실한 자동 세팅은 즉시 반영한다.
3. 토스트나 요약 문구로 무엇이 적용됐는지 보여준다.
4. 필요하면 바로 되돌릴 수 있게 한다.
5. 업로드 후에는 추천 세팅을 한 번에 검토하고 적용할 수 있게 묶어준다.

예시:

- `생활비 입출금 계좌로 보고 관련 카테고리 14개를 연결했어요. 되돌리기`
- `카드 납부용 계좌 후보로 설정했어요. 되돌리기`

## 흐름 페이지와의 연결

- `흐름` 페이지는 사람 간 정산이 아니라 월말 돈 흐름 정리 단계다.
- 자산/카테고리 세팅과 이번 달 수입 입력, 카드명세 정리가 끝난 뒤 들어간다.
- 카테고리별 연결 계좌를 기준으로 카드값 계좌에 얼마를 이체해야 하는지 정리해서 보여준다.
- 사용자가 각 이체 건의 실행 여부를 확인하면 이번 달 흐름 정리가 끝난다.

## 우선순위

1. `card_payment` 자동 세팅 정리
2. `salary` 자동 세팅 정리
3. `자동 적용 + 되돌리기` 테스트 보강
4. 업로드 이후 `추천 세팅 한 번에 적용` 흐름 정리

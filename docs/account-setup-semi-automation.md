# 이음 설정 반자동화 구상

## 배경

이 작업의 목적은 사용자의 진입장벽을 낮추고, 기본 세팅이 많아 보여서 느껴지는 부담을 줄이는 데 있다.
사용자가 설정 화면을 오래 헤매기 전에 앱이 먼저 높은 확률의 기본값을 채워서, 빠르게 "쓸 수 있는 상태"를 만들어 주는 것이 핵심이다.

## 목표

- 사용자가 초반에 직접 결정해야 하는 항목 수를 줄인다.
- 설정 화면에 들어가기 전에도 기본 흐름이 동작하도록 만든다.
- 자동화가 틀릴 수 있는 부분만 짧고 가볍게 확인받는다.
- 설정은 "해야 하는 일"이 아니라 "이미 거의 끝난 상태를 확인하는 일"처럼 느껴지게 만든다.

## 원칙

1. 확실한 건 자동 적용한다.
2. 애매한 것만 짧게 확인받는다.
3. 막는 팝업보다 `자동 적용 + 되돌리기`를 우선한다.
4. 같은 타입의 두 번째 계좌부터는 조용히 지나가고, 필요할 때만 제안한다.

## 현재 구현 상태

### 1. 생활비 통장 자동 연결

- 첫 `daily` 계좌가 생성되거나 기존 계좌가 처음 `daily` 타입으로 바뀌면 자동 연결을 실행한다.
- 연결 대상은 다음 카테고리들이다.
  - `주거/고정비`
  - `생활비`
  - `생필품`
  - `교통/차량`
  - `의료/건강`
  - `가족/관계`
  - `데이트/여행`
  - `세금/공과`
  - `기부금`
- 이미 같은 타입의 계좌가 하나라도 있었다면 두 번째부터는 아무 자동 이벤트도 발생하지 않는다.

### 2. 대출금 통장 자동 연결

- 첫 `loan` 계좌가 생성되거나 기존 계좌가 처음 `loan` 타입으로 바뀌면 자동 연결을 실행한다.
- 연결 대상은 `대출/부채` 그룹 아래 카테고리들이다.
- 이미 같은 타입의 계좌가 하나라도 있었다면 두 번째부터는 아무 자동 이벤트도 발생하지 않는다.

## 다음 자동화 룰맵

### 1. usageType 기반 자동 세팅

- 첫 `daily` 계좌
  - 생활 계열 카테고리 자동 연결
- 첫 `loan` 계좌
  - `대출/부채` 계열 카테고리 자동 연결
- 첫 `card_payment` 계좌
  - 카드 납부계좌 후보 기본값으로 우선 사용
- 첫 `salary` 계좌
  - 월수입 기준 계좌 후보로 지정
- 첫 `shared` 계좌
  - 공동지출, 맺음 흐름의 기본 계좌 후보로 지정
- 같은 타입의 두 번째 계좌
  - 자동 적용하지 않음

### 2. 자동 적용 방식

- 가능하면 즉시 적용한다.
- 적용 직후에는 결과를 짧게 알려준다.
- 사용자가 원하지 않으면 바로 되돌릴 수 있게 한다.

예시:

`생활비 통장을 연결해서 관련 카테고리 14개를 자동 연결했어요. 되돌리기`

### 3. 나중에 보여줄 제안

- 처음에는 조용히 자동 적용한다.
- 이후 미연결 항목이 남았을 때만 가벼운 제안을 띄운다.

예시:

`생활비 관련 미연결 카테고리 3개를 이 계좌에 연결할까요?`

## 추천 UX 흐름

1. 사용자, 계좌, 카드 생성
2. `usageType` 기반 자동 세팅 즉시 반영
3. 업로드 이후 추천 세팅 묶음 제안
4. 사용자는 마지막 확인만 진행

## 업로드 이후 추천 세팅 묶음 예시

- 카드 납부계좌 연결
- 생활비 통장 연결
- 급여 통장 지정

## 우선순위

1. `usageType`별 자동 세팅 룰맵 정리
2. `자동 적용 + 되돌리기` 토스트 추가
3. 업로드 이후 `추천 세팅 한 번에 적용` 흐름 추가

## 구현 메모

- 자동화는 "모든 걸 묻는 흐름"이 아니라 "거의 끝난 상태를 보여주는 흐름"이어야 한다.
- 잘못 붙을 가능성이 낮은 기본값은 팝업보다 자동 적용이 더 적합하다.
- 팝업은 막는 용도보다, 애매한 연결을 짧게 확인받는 용도로만 쓰는 편이 낫다.
- 이 문서는 현재 상태를 바탕으로 다음 반자동화 작업을 이어갈 때 기준 문서로 사용한다.
